# 矩阵助手 · Ops 管理端前端重构设计

**日期：** 2026-07-13  
**状态：** 已实施完成（计划见 `docs/superpowers/plans/2026-07-13-matrix-assistant-ops-web.md`）  
**范围：** 用 Vue 3 SPA **一次性整站替换**现有 Jinja `/ops` 界面；面向完全不懂计算机的运营新手；界面语言简体中文；电脑优先并预留移动端；**多平台数据模型预留（本阶段只做小红书，不实现抖音）**。  
**上位：** `2026-07-12-xhs-cloud-management-design.md`、`2026-07-13-ops-publish-comments-inbox-design.md`  
**非目标：** 本规格不实现端上 Accessibility、不引入 LLM 生成文案、不做多租户/多运营角色权限控制台（权限字段仍预留）、不做原生 App、**不实现抖音端上自动化与抖音业务页**。

| 项 | 选择 |
| --- | --- |
| 主要用户 | 运营人员（发帖 / 回评论 / 回私信）；少碰技术细节，但仍要完整设备与任务管理 |
| 首页 | 「今日待办」清单，点一项去做 |
| 技术异常展示 | 待办用人话提示；**系统**区保留完整设备 / 任务 / 告警页 |

## 1. 目标与成功标准

### 1.1 目标

把「矩阵助手」做成运营每天打开就能干活的中文管理端：不认识 `task_id` / `error_code` 也能发笔记、回评论、回私信、看线索；需要排障时能在「系统」区看设备与任务。

### 1.2 成功标准（验收）

1. 登录后默认进入「今日待办」，文案与导航均为简体中文。  
2. 运营可在 SPA 内完成：上传并立即发布（或排期执行）→ 同步并回复评论 → 同步并回复私信 → 标记线索。  
3. 发帖成功后默认自动产生互动链；重要笔记可手动指定引导评论账号后执行；可停止链。  
4. 账号 / 设备 / 任务 / 告警页功能不低于现 Jinja + 既有 API 能力，且状态有人话摘要。  
5. 旧 `/ops/*.html` Jinja 路由下线或 301 到 SPA 等价路由；无双前端长期并存。  
6. 布局采用响应式断点变量与可触控间距约定，但本阶段只保证桌面（≥1280px）完整体验。  
7. 账号 / 内容 / 任务等 API 与前端类型带 `platform` 字段；默认与筛选均为小红书；界面不出现可用的「抖音」业务入口（可在账号详情只读展示平台标签「小红书」）。

## 2. 信息架构与导航

左侧固定导航 + 右侧工作区；顶栏：产品名「矩阵助手」、帮助（「遇到问题怎么办」）、退出。  
本阶段顶栏可不做平台切换器；待办与列表的数据请求固定 `platform=xhs`（或服务端默认）。预留：日后顶栏「平台」筛选只影响列表作用域，不改导航结构。

### 2.1 日常工作

| 导航名 | 路由建议 | 作用 |
| --- | --- | --- |
| 今日待办 | `/` | 默认首页，待办聚合 |
| 发笔记 | `/publish` | 发布向导与排期执行 |
| 评论 | `/comments` | 同步 / 列表 / 回复 |
| 私信 | `/inbox` | 会话列表与详情回复 |
| 线索 | `/leads` | 意向筛选与标记 |
| 内容库 | `/contents` | 素材、话术、名片图 |
| 互动链 | `/engagement` | 链列表、自动结果、手动编排、停链 |

### 2.2 系统

| 导航名 | 路由建议 | 作用 |
| --- | --- | --- |
| 账号 | `/accounts` | 角色、绑定、暂停/恢复 |
| 设备 | `/devices` | 在线、无障碍、网络摘要 |
| 任务 | `/tasks` | 执行记录、失败码详情 |
| 告警 | `/alerts` | 停机类与绑定异常汇总 |

### 2.3 人话原则

- 按钮：同步评论、发送回复、立即发布、执行排期、停止互动。  
- 失败：优先「手机还没准备好，请稍后重试」等；任务详情才展示 `error_code`。  
- 禁止在主流程展示 UUID 作为唯一标识；列表可用「标题摘要 + 时间」。

## 3. 关键页面与交互

### 3.1 今日待办

数据源聚合（只读视图 + 跳转）：

| 类型 | 来源示意 | 主按钮 |
| --- | --- | --- |
| 待回复私信 | 未读 / 待处理会话 | 去回复 |
| 待回复评论 | 已同步评论中待运营处理（首期可先「有新评论的笔记」） | 去回复 |
| 待发布 / 可执行排期 | ScheduleItem 待执行 | 去发布 |
| 待跟进线索 | Lead 开放状态 | 去处理 |
| 异常 | 设备 offline / a11y 掉绑、近期 failed 副作用任务 | 查看原因 |

支持按账号筛选。点击携带深链 query（`account_id`、`note_id`、`thread_id` 等）打开业务页。

### 3.2 发笔记

向导步骤：选发帖账号（仅 `PUBLISHER_*`）→ 选已有素材或上传新建 → 预览标题正文 → **立即发布** 或 **加入排期** → 结果页展示人话进度。

选项：「这是重要笔记，发布后手动编排互动」——勾选则发帖成功跳转互动链编排页，**不**自动开链；未勾选则走默认自动开链。

### 3.3 评论

选账号与笔记 →「同步最新评论」→ 列表 → 对单条填写回复 → 发送。进度组件轮询任务状态直至成功/失败。

### 3.4 私信

会话列表 → 详情消息流 → 回复区仅允许话术模板或名片图（合规拒绝明文联系方式并当场提示）。支持「同步私信」。

### 3.5 线索

列表筛选 intent / handoff；从私信详情可一键标记；待办可直达。

### 3.6 内容库

图文素材上传与列表；敏感词结果对人展示「通过 / 需改写」；话术模板与名片图 CRUD，供发帖与私信选用。

### 3.7 互动链（见第 4 节）

### 3.8 账号 / 设备 / 任务 / 告警

- **账号：** 角色中文名（主号 / 矩阵号 / 互动号）、**平台标签（本阶段固定「小红书」）**、绑定设备、暂停与恢复。  
- **设备：** 在线、无障碍是否就绪、Wi‑Fi/蜂窝摘要、最近心跳（相对时间）；设备本身跨平台复用，**账号绑定决定跑哪个 App**。  
- **任务：** 完整列表与详情；副作用任务禁止「一键重做」；提供「我知道了 / 关闭」。  
- **告警：** 无障碍掉绑、离线、风控停机等；链到设备或任务。

### 3.9 登录与帮助

保留共用运营 Token/Session 登录（与现网一致）。帮助页用 5～8 条步骤说明：发帖、回评论、回私信、手机红灯怎么办。

## 4. 互动链：自动默认 + 手动编排

### 4.1 角色约束（不变）

- 仅 `ENGAGER` 账号可发引导评论（`post_comment`）。  
- `PUBLISHER_*` 可对引导评做回复（`reply_comment`）。  
- API / 编排器双门禁；ENGAGER 不可发帖。

### 4.2 默认自动

发帖任务 `effect_committed` 且笔记未标记「手动编排」时：

1. 从状态正常、未暂停、有配额的 `ENGAGER` 池中按策略选 N 个账号（N 可配置，默认与现网托评条数一致）。  
2. **选号策略（首期）：** 伪随机 + 避开近期过密使用同一号（简单公平轮转即可）；不要求运营理解算法。  
3. 创建 `EngagementChain` 与子任务；失败停链并告警，待办出现人话项。

### 4.3 手动编排（重要笔记）

入口：发帖向导勾选，或互动链页对某笔记「手动编排」。

编排 UI：

1. 展示目标笔记摘要。  
2. 多选可用互动号；可调顺序；显示「将使用这些账号依次引导评论」。  
3. 可选：是否在引导后自动安排主号/发帖号回复（沿用现链配置）。  
4. 确认后创建链；未确认不创建。  
5. 进行中可「停止互动」。

### 4.4 列表与透明度

链列表展示：笔记标题摘要、模式（自动/手动）、参与账号昵称或账号中文备注、状态、时间。新手不需填写 playbook 名。

### 4.5 后端缺口（本前端规格依赖）

若现 API 仅有「发帖后固定自动开链」，需扩展（可与 SPA 同期）：

- 发帖/内容参数或笔记标记：`engagement_mode=auto|manual`。  
- `POST /engagement-chains`（或等价）：body 含 `note_id` / `publish_task_id`、`engager_account_ids[]`（手动）、可选顺序。  
- 自动路径保留现有 Orchestrator；手动路径跳过随机选号。  
- `GET /engagement-chains` 返回模式与账号列表供 UI。

## 5. 前端技术架构

### 5.1 工程位置

新建仓库内目录：`ops-web/`（Vue 3 + Vite + TypeScript + Vue Router + Pinia）。  
UI 组件库建议 **Element Plus**（中文友好、表格/表单/向导齐全）；主题变量集中，避免日后移动端重写视觉。

### 5.2 与 Control Plane 关系

```text
浏览器 → 矩阵助手 SPA（静态资源）
        → HTTPS /api/v1/*  （FastAPI，现有鉴权）
        → （可选）/ops/login 仅保留签发 session 的薄接口，或改为 SPA 登录调 API
```

- 鉴权：**沿用现网 Ops Cookie Session**（登录成功写 Cookie；SPA 请求 `credentials: 'include'`）。若部署跨域，再改为同域反代或补 Bearer；默认同域托管避免跨域。  
- 开发：Vite proxy 到 control-plane（含 Cookie）。  
- 生产：Nginx/Caddy 或 FastAPI `StaticFiles` 托管 `ops-web/dist`，`/api` 反代后端；**`/ops` 与 `/` 由 SPA history fallback 处理。**

### 5.3 替换策略（一次性切换）

1. SPA 在功能清单（第 2～4 节）达到验收后合并。  
2. 切换日：静态资源上线；Jinja `web/routes.py` HTML 路由改为重定向到 SPA 路径或删除。  
3. 保留 API 与 Ops Token；不保留长期 Jinja 双轨。  
4. Docker：control-plane 镜像增加前端 build 阶段或独立 `ops-web` 构建 job，产物拷入静态目录（符合 Docker-only 开发约束）。

### 5.4 移动端预留（本阶段不实现）

- 使用弹性布局与断点 CSS 变量（`--bp-md` 等）。  
- 导航抽象为可切换「侧栏 / 底栏」的 shell 组件，桌面只用侧栏。  
- 不单独做移动视觉稿；禁止写死仅桌面可用的绝对定位工作流。

### 5.5 多平台预留（本阶段不实现抖音）

**现状诚实约束：** 端上 Agent 目前绑定小红书包名与页面；抖音不能靠改配置上线，需日后单独做端上适配。云端「设备 / 账号 / 任务 / 内容 / 互动 / 私信」模式可复用。

**本阶段必须落地的预留（防返工）：**

| 位置 | 约定 |
| --- | --- |
| 数据模型 | `Account`、`Content`、`Note`/`published_notes`、`EngagementChain`、`InboxThread`、`Task`（或 params）带 `platform`；枚举至少含 `xhs`，预留 `douyin`。缺省与迁移默认 `xhs`。 |
| API | 列表接口支持 `platform` 查询参数；写接口校验账号与内容平台一致，禁止跨平台混用。 |
| 前端类型 | `Platform = 'xhs' \| 'douyin'`；store / 筛选预留；本阶段常量 `ACTIVE_PLATFORMS = ['xhs']`，UI 不渲染未开通平台。 |
| 文案 | 业务词尽量用中性词（「作品/笔记」「评论」「私信」）；小红书特有说明放帮助或括号，避免全文写死「仅小红书」导致改抖音时大面积替换。发帖主按钮可用「发笔记」（xhs）；平台文案表按 `platform` 取词，为抖音预留「发视频」等键位但不启用。 |
| 端上（远期） | 设备执行按账号 `platform` 选择 App 适配器；本规格**不**设计抖音 PageHint / Playbook。 |
| 互动链 | 自动/手动选号仅在同一 `platform` + 同角色池内进行。 |

**本阶段明确不做：** 抖音登录/发布/评论/私信自动化、抖音合规规则专项、Ops 里可切换到抖音工作台。

### 5.6 状态与任务进度

共用 `useTaskPoll(taskId)`：提交副作用后展示进度条；成功/失败 toast + 待办刷新。禁止静默失败。

### 5.7 国际化

首期仅 `zh-CN` 文案资源文件；不接英文包。代码与路由可用英文标识。

## 6. 错误处理与新手文案映射

| 系统信号 | 界面人话（示例） | 下一步 |
| --- | --- | --- |
| `A11Y_DOWN` / a11y_bound=false | 手机辅助功能还没就绪 | 打开「设备」查看 |
| offline / 心跳超时 | 手机好像不在线 | 检查手机网络与 Agent |
| `NAV_*` / `WRONG_PAGE` | 手机还停在别的页面，这次没做成 | 重试；仍失败联系管理员 |
| 合规拒绝 | 回复里不能出现微信号/手机号 | 改用话术或名片图 |
| 敏感词拒绝 | 文案需要改写后才能用 | 回内容库修改 |
| 角色门禁 | 这个账号不能发笔记 | 换主号/矩阵号 |

审计：关键写操作记 AuditLog（谁、何时、对哪个账号）。

## 11. 决策摘要（给实现计划用）

1. 产品名「矩阵助手」，Vue 3 + Element Plus SPA，替换全部 Ops HTML。  
2. 导航：日常工作 7 项 + 系统 4 项；首页今日待办。  
3. 互动链默认自动随机/轮转选 ENGAGER；重要笔记手动多选排序。  
4. 电脑优先 CSS 预留移动；Docker 构建前端产物随 CP 或独立服务托管。  
5. 对外一次切流；对内按 W0–W5 实现。  
6. **多平台：模型/API/前端预留 `platform`；本阶段只开通小红书；抖音暂不实现。**
